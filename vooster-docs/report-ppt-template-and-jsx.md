# PPT 템플릿과 대시보드 시안(.jsx) 반영 방법

## 제공하신 소스

1. **슬라이드 템플릿 이미지**  
   표지·노출량 요약·배너·문서 끝 슬라이드의 **디자인 이미지** (검정 배경, 네온 라인, kt nasmedia 로고, 플레이스홀더 {광고주명}, {년.월.일} 등).

2. **지하철_역사별_예상_노출량_대시보드_시안.jsx**  
   React + Recharts 기반 **노출량 대시보드 시안**.  
   - 공공데이터 기반 추정 / 배너 노출량 탭  
   - KPI 카드 4개 (역/호선, 일평균 유동인구, 게재 기간, 예상 총 노출량)  
   - 하이라이트 (피크 시간대, 2순위)  
   - 시간대별 예상 노출량 표 (순위·비중·미니 바)  
   - 시간대별 막대그래프, 시간대 비중 도넛차트  
   - 주석(추정치·출처)

## PPT에 반영되는 방식

- **.jsx** 는 웹 대시보드용이라 PPT 안에서 그대로 실행되지 않습니다.  
- PPT에서는 **slide-02-summary** 에 쓰는 **이미지**가 위 시안과 같은 구도(카드·표·그래프 배치)로 되어 있으면, 코드가 그 위에 **숫자만** 덮어씁니다.
- 따라서:
  1. 시안(.jsx)으로 노출량 대시보드 화면을 **한 장의 이미지로 렌더/캡처**하고,  
  2. 그 이미지를 `public/report-ppt-templates/slide-02-summary.png` (또는 `.png.jpg`) 로 저장하면,  
  3. PPT 생성 시 해당 이미지가 노출량 요약 슬라이드 **배경**으로 쓰이고, 역/호선·일평균·게재기간·총노출량·시간대별 표 등은 코드가 **텍스트/표**로 그 위에 그립니다.

## 템플릿이 “전혀 반영 안 됨”일 때 확인할 것

1. **파일 위치·이름**  
   제공하신 표지·요약·배너·끝 **이미지 파일**이  
   `public/report-ppt-templates/` 아래에  
   `slide-01-cover.png` 또는 `slide-01-cover.png.jpg`,  
   `slide-02-summary.png` 또는 `slide-02-summary.png.jpg`,  
   `slide-03-banner.png` 또는 `slide-03-banner.png.jpg`,  
   `slide-04-End.png` 또는 `slide-04-End.png.jpg`  
   로 들어가 있는지 확인하세요.

2. **실행 경로**  
   Python 스크립트는  
   - 프로젝트 루트의 `public/report-ppt-templates`  
   - 또는 **현재 작업 디렉터리**의 `public/report-ppt-templates`  
   에서 위 파일명을 찾습니다.  
   프로젝트 루트(`d:\cursor\admate_vision_alpha`)에서 실행했는지 확인하세요.

3. **환경 변수(선택)**  
   템플릿 폴더를 다른 경로로 쓰려면  
   `REPORT_PPT_TEMPLATES_DIR=원하는_폴더_절대경로` 로 지정할 수 있습니다.

요약: **제공하신 템플릿/소스는 “이미지 파일을 정해진 이름으로 public/report-ppt-templates 에 두는 것”과 “slide-02는 .jsx 시안과 같은 구도의 이미지를 쓰는 것”으로 반영됩니다.**
